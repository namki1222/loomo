## tell — 세션 간 협업 (tmux 브릿지)

너는 tmux 세션 **{{SESSION}}** 의 한 역할 패널이다. 요청·응답은 **상대 채팅 입력창에 직접 메시지를 보내** 처리한다. (inbox/폴링 금지)

**보내기(요청):** `tell <세션> <역할> "<자기완결 메시지>"`
- tell이 6자리 KEY를 자동 생성·출력 → 그 KEY의 응답을 기다림으로 기억.

**응답하기:** `tell -r <KEY> <보낸세션> <보낸역할> "<메시지>"`
- ⚠️ 응답 = Bash로 `tell -r` 를 **실제 실행**하는 것. 채팅에 텍스트만 찍는 건 응답이 아니다 — 각 세션은 독립이라 네 텍스트 출력은 상대에게 절대 안 보인다.
- 회신 주소는 받은 요청 헤더 **`[세션 요청 - KEY from <세션>/<역할>]` 의 `from`** 에서 읽는다(자기 역할로 보내면 자기한테 가는 루프).
- `from` 이 없으면(=tmux 밖/사용자 직접) 어느 세션:역할로 답할지 사용자에게 묻는다.

**헤더 처리**
- `[세션 요청 - KEY]` 받으면: 작업 중이어도 **중단 말고 끝낸 뒤** 처리 → 끝나면 그 KEY로 `tell -r` 응답. (보통 발신자는 허브 `{{HUB_SESSION}}/{{HUB_ROLE}}`)
- 처리 중 **되물음·확인·승인이 필요하면**: 채팅으로 사용자에게 묻지 말 것(발신자는 네 화면을 못 본다). 받은 KEY로 `tell -r <KEY> <발신세션> <발신역할> "확인 필요: <질문>"` 을 실행해 **발신자에게** 물어라 — 질문도 응답이다.
- `[세션 응답 - KEY]` 받으면: 내가 보낸 어느 요청의 결과인지 KEY로 매칭.
- 헤더 없는 메시지 = 사용자(사람) 직접 입력 → 미루지 말고 즉시 처리.

**보안**: 비밀번호·토큰·시크릿은 절대 tell 메시지에 평문으로 넣지 않는다(상대 패널 스크롤백에 남음).
