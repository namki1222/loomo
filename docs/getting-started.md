# 처음부터 따라하는 설치 가이드 (`tell init`)

> tmux를 한 번도 안 써봤어도 됩니다. 이 문서의 명령을 **위에서부터 순서대로 복사-붙여넣기**만 하면, 약 10분 뒤 두 Claude가 서로 대화(핑퐁)하는 것까지 확인합니다.

완성하면 이런 그림이 됩니다:

```
┌─ 터미널 한 화면 (세션 "demo") ───────────────────────────┐
│  [api 패널]                │  [web 패널]                │
│  Claude Code 실행 중        │  Claude Code 실행 중        │
│                            │                            │
│  둘이 tell 명령으로 서로 요청하고 응답함                    │
└────────────────────────────────────────────────────────┘
```

---

## 0. 준비물 설치

터미널을 열고 아래를 하나씩 확인하세요. **이미 있으면 건너뛰면 됩니다.**

**① tmux** — 터미널 화면을 여러 칸으로 나눠주는 프로그램. 이 도구의 기반입니다.

```bash
tmux -V        # "tmux 3.x" 가 나오면 이미 설치됨
```

없다면:

```bash
# macOS
brew install tmux

# Ubuntu/Debian 계열 Linux
sudo apt install tmux
```

**② Node.js / npm** — 설치 채널로만 씁니다 (실행에는 필요 없음).

```bash
npm -v         # 버전이 나오면 OK
```

없다면 https://nodejs.org 에서 LTS 버전을 설치하세요.

**③ Claude Code** — 각 패널 안에서 실제로 일할 AI입니다.

```bash
claude --version    # 버전이 나오면 OK
```

없다면:

```bash
npm install -g @anthropic-ai/claude-code
claude              # 첫 실행 시 로그인 안내를 따라가세요
```

## 1. claude-tell-bridge 설치

```bash
npm install -g claude-tell-bridge
```

설치가 끝나면 환경 점검:

```bash
tell doctor
```

```
════════════════════════════════════════════════════
   🔗 claude-tell-bridge — doctor · 환경 점검
════════════════════════════════════════════════════
✅ tmux 3.6b
✅ claude 2.1.198
ℹ️  tmux 밖 — 발신자(from) 헤더 없이 전송됨
ℹ️  워크스페이스 설정 없음 — 'tell init' 또는 'tell adopt'로 생성
✅ 패키지 리소스 정상
```

`✅ tmux` 와 `✅ 패키지 리소스 정상` 두 줄만 보이면 진행 가능합니다. (`ℹ️` 줄들은 아직 설정 전이라 뜨는 안내일 뿐, 정상입니다.)

## 2. 연습용 프로젝트 폴더 만들기

실제 프로젝트가 있다면 그 경로를 써도 되지만, 처음이니 연습용 폴더 두 개로 시작합시다:

```bash
mkdir -p ~/tell-demo/api ~/tell-demo/web
```

- `~/tell-demo/api` → "백엔드 담당" Claude가 상주할 곳
- `~/tell-demo/web` → "프론트 담당" Claude가 상주할 곳

## 3. `tell init` — 셋업 마법사

```bash
tell init
```

마법사가 질문을 던집니다. **아래 표시된 대로 입력**하세요. (언제든 `Ctrl+C`로 중단해도 그때까지 입력분은 저장됩니다.)

### [1/2] 허브 세션 — 일단 건너뜁니다

```
▶ [1/2] 총괄 관리(허브) 세션 — 선택사항
  허브란? 여러 프로젝트를 대신 지휘해주는 '비서' Claude 세션입니다.
  ...
총괄 관리 세션을 만들까요? [y/N]: ⏎ (그냥 엔터)
```

허브는 "여러 프로젝트를 대신 지휘하는 비서"인데, 처음엔 필요 없습니다. 나중에 `tell hub` 한 번으로 언제든 추가할 수 있으니 **그냥 엔터**로 건너뛰세요.

### [2/2] 프로젝트 등록 — 핵심 단계

개념은 딱 두 개입니다: **프로젝트(세션) 하나** 안에 **패널(역할) 여러 개**. 패널 하나 = Claude 한 명입니다.

```
▶ [2/2] 프로젝트 등록

프로젝트 이름 = 세션 이름 (빈 값 = 완료): demo

  추가할 패널(역할) 이름 (빈 값 = 이 프로젝트 완료): api
  'api' 담당 디렉터리: ~/tell-demo/api
  ✔ 패널 추가 완료 — demo · api

  추가할 패널(역할) 이름 (빈 값 = 이 프로젝트 완료): web
  'web' 담당 디렉터리: ~/tell-demo/web
  ✔ 패널 추가 완료 — demo · web

  추가할 패널(역할) 이름 (빈 값 = 이 프로젝트 완료): ⏎ (엔터)
✔ 프로젝트 'demo' 구성 완료

프로젝트 이름 = 세션 이름 (빈 값 = 완료): ⏎ (엔터)
```

입력 순서를 정리하면: `demo` → `api` → `~/tell-demo/api` → `web` → `~/tell-demo/web` → 엔터 → 엔터. 끝.

> 이 단계에서 각 디렉터리의 `CLAUDE.md`에 협업 규약이 자동으로 삽입됩니다 — 받는 쪽 Claude가 "응답은 어떻게 보내야 하는지" 아는 근거입니다. 마법사가 알아서 하니 신경 쓸 것 없습니다.

## 4. 세션 띄우기

```bash
tell ws demo
```

이 한 줄이: 세션 `demo` 생성 → `api`/`web` 패널로 화면 분할 → 각 패널에서 Claude Code 자동 실행 → 그 화면으로 접속, 까지 다 합니다.

**화면이 두 칸으로 나뉘고 양쪽에 Claude Code가 뜨면 성공입니다.**

### 처음 보는 분할 화면, 이것만 알면 됩니다

| 하고 싶은 것 | 키 |
|---|---|
| 옆 패널로 이동 | `Ctrl+b` 누렀다 떼고 → 화살표 키 (또는 마우스로 클릭*) |
| 이 화면에서 잠시 나가기 (세션은 계속 살아있음) | `Ctrl+b` 누렀다 떼고 → `d` |
| 다시 들어가기 | 터미널에서 `tell ws demo` |

\* 마우스 클릭이 안 되면 tmux 마우스 모드가 꺼진 것 — `~/.tmux.conf`에 `set -g mouse on` 한 줄을 추가하면 됩니다 (필수는 아님).

## 5. 첫 핑퐁 — 두 Claude가 서로 대화하는지 확인

`api` 패널을 클릭(또는 `Ctrl+b` → 화살표)해서 이동한 뒤, 그 패널의 Claude에게 **그냥 말로** 부탁하세요:

```
web한테 핑퐁 테스트 보내고 응답 오는지 확인해줘
```

`tell` 명령을 직접 외울 필요 없습니다 — 3단계에서 자동 삽입된 규약 덕에 Claude가 알아서 합니다:

1. `api`의 Claude가 스스로 `tell demo web "..."`을 실행 → `web` 패널 입력창에 메시지가 **직접 타이핑**됩니다
2. `web`의 Claude가 메시지를 받고, 규약에 따라 `tell -r`로 응답을 보냅니다
3. 몇 초 뒤 `api` 패널에 이렇게 돌아옵니다:

```
[세션 응답 - 3f9a1c from demo/web] 퐁 정상
```

**이 줄이 보이면 설치 끝.** 이제 두 세션은 서로에게 일을 시킬 수 있습니다.

## 6. 실전 사용 예

핑퐁의 "테스트 메시지" 자리에 실제 작업을 넣으면 그게 실전입니다. `api` 패널의 Claude에게:

```
주문 목록 API에 페이지네이션 추가해줘. 끝나면 web한테
tell로 스키마 바뀐 거 알려주고 프론트 반영시켜줘
```

그러면 `api` Claude가 작업 후 스스로 `tell demo web "..."`을 보내고, `web` Claude가 반영한 뒤 응답까지 보냅니다 — 사람이 복사-붙여넣기로 중계할 필요가 없어집니다.

메시지를 직접 보낼 땐 **자기완결**로 쓰는 게 요령입니다: 무엇을·어디를·어떻게 + "끝나면 받은 키로 응답해줘"까지 한 통에.

## 7. 자주 쓰는 명령 요약

```bash
tell list          # 주소록 — 지금 말 걸 수 있는 세션:역할 목록
tell ws demo       # 세션 접속 (없으면 생성부터)
tell ws            # 설정된 워크스페이스 목록
tell rm demo       # 세션 정리 (tmux 종료 + 설정 제거, 각각 확인함)
tell doctor        # 뭔가 이상할 때 환경 점검
```

## 막혔을 때

| 증상 | 해법 |
|---|---|
| `tell: command not found` | npm 전역 bin이 PATH에 없음. `npm bin -g` 경로를 PATH에 추가하거나 터미널 재시작 |
| 패널에 Claude 대신 빈 셸만 뜸 | 그 디렉터리가 없거나 `claude`가 PATH에 없음. `tell doctor`로 확인 |
| 메시지는 갔는데 응답이 안 옴 | 받는 쪽이 규약을 아직 안 읽은 것. `tell demo web "CLAUDE.md를 Read로 읽고 받은 키로 tell -r 응답해줘"` 한 통이면 해결 |
| `[tell] 대상 패널 없음: demo:web` | 세션/역할 이름 오타. `tell list`로 실제 주소 확인 |
